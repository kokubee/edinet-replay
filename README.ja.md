# EDINET Replay（日本語）

> 正本は英語版 [README.md](README.md) です。本ファイルは補助的な日本語訳で、内容が食い違う場合は英語版が優先します。

日本の EDINET（金融庁の電子開示システム）提出書類について、**原典への追跡可能性を維持した**再現可能な抽出ワークフローを開発するオープンソースプロジェクトです。現状はスキーマと仕様の段階です。

公式の提出書類パッケージの保全、XBRL / Inline XBRL の [Arelle](https://arelle.org/) による解決、原典へ追跡可能な機械可読表現の生成——これらのためのバージョン付きスキーマと再現性契約を定義しています。

想定利用者は、データエンジニア・クオンツ研究者・XBRL実装者・ESGデータ利用者・学術研究者・データベンダー・機関投資家のデータ担当者です。一般個人投資家が直接使うことは想定していません。

> **現状: pre-alpha。** 現在提供しているのはバージョン付き JSON Schema・再現性仕様・検証用 fixture です。取得クライアントと Arelle ベースの実装は開発中で、本番利用可能な状態ではありません。

## このプロジェクトの目的

EDINET は公開情報ですが、再現可能な利用は難しいままです（XBRL/iXBRL 構造の複雑さ、タクソノミの年度変化、抽出時の context/unit/dimension/来歴の欠落、正規化DBの変換根拠の不透明さ）。

本プロジェクトは**財務的な解釈ではなく忠実な再現**を第一に据えます。名寄せ・正規化・比較可能性判定・投資分析は別の下流レイヤーに置きます。

## やらないこと

- 異なるXBRL概念を共通の経済的概念とみなして統合しない
- データの経済的意味を正規化・訂正・保証しない
- 企業間・期間間・会計基準間の比較可能性を保証しない
- 投資助言をしない
- EDINET / 金融庁の公式製品ではない

> 本プロジェクトは独立したオープンソースプロジェクトであり、金融庁と提携・出資・承認関係にありません。
